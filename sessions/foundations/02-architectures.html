Agent Architectures Overview

Explore reactive, deliberative, hybrid, and BDI architectures with a comparison table and design guidance.

30 min Foundations Hands-on Lab 5 Quiz Questions

Why Architecture Matters

Before writing a single line of agent code, you need to choose an architecture — the fundamental pattern that determines how your agent perceives, reasons, and acts. The wrong architecture choice leads to agents that are too slow, too rigid, too unpredictable, or too expensive to run. The right choice depends on your task's complexity, latency requirements, and environment dynamics.

In classical AI and robotics, agent architectures were studied for decades before large language models arrived. Understanding these foundations helps you reason about LLM-based agents with much greater precision.

Four major families: Reactive (stimulus-response), Deliberative (plan then act), Hybrid (layered reactive + deliberative), and BDI (Beliefs-Desires-Intentions). Modern LLM agents blend elements of all four.

1. Reactive Architectures

Reactive architectures, pioneered by Rodney Brooks at MIT in the 1980s, reject the idea that an agent needs an explicit world model. Instead, the agent connects perception directly to action through a set of situation-action rules. Brooks called his layered version the Subsumption Architecture.

Subsumption Architecture (Brooks, 1986)

Behaviours are layered in a priority hierarchy. Lower layers (e.g., "avoid obstacles") can subsume (override) higher layers (e.g., "move toward goal"). The agent responds purely to current sensor input — no planning, no memory of the past.

Pros: Extremely fast response, robust to sensor noise, easy to implement.
Cons: Cannot handle tasks requiring planning, learning, or goal reasoning.

Modern analogy: A retrieval-augmented chatbot with rule-based routing — fast, no planning.

# Reactive agent: direct stimulus → response mapping
class ReactiveAgent:
    def __init__(self):
        # Priority-ordered list of (condition, action) pairs
        self.rules = [
            (lambda obs: "error" in obs.lower(),   self.handle_error),
            (lambda obs: "urgent" in obs.lower(),  self.escalate),
            (lambda obs: "question" in obs.lower(),self.answer_question),
            (lambda obs: True,                     self.default_response),
        ]

    def act(self, observation: str) -> str:
        for condition, action in self.rules:
            if condition(observation):
                return action(observation)

    def handle_error(self, obs):   return f"ERROR HANDLER: {obs}"
    def escalate(self, obs):       return f"ESCALATING: {obs}"
    def answer_question(self, obs):return f"ANSWER: {obs}"
    def default_response(self, obs):return f"DEFAULT: {obs}"

2. Deliberative Architectures

Deliberative (or "classical") architectures build an explicit model of the world, formulate a plan, and then execute it. This approach dominated AI research through the 1990s under the banner of symbolic AI. The STRIPS planner and SOAR cognitive architecture are canonical examples.

Deliberative Cycle: Model → Plan → Execute

World Model: The agent maintains a symbolic representation of the environment (predicates, facts, states).
Goal: A desired state expressed in the same symbolic language.
Planner: Searches for a sequence of actions that transforms the current state into the goal state.
Executor: Carries out the plan step by step.

Pros: Can handle complex, long-horizon tasks; produces explainable plans.
Cons: Slow (planning is expensive); brittle when the world deviates from the model.

3. Hybrid Architectures

Hybrid architectures, also called layered architectures, combine reactive and deliberative components. The reactive layer handles time-critical responses while the deliberative layer plans longer-horizon behaviour. The two layers communicate through a shared state.

Three-Layer Architecture

Reactive Layer (bottom): Sensor data → immediate action. Sub-second response.
Executive Layer (middle): Sequences reactive behaviours, monitors plan execution.
Deliberative Layer (top): Long-term planning, goal management, world modeling.

Modern analogy: A LangGraph agent where tool-call nodes are reactive, a supervisor node is executive, and an LLM planner node is deliberative.

4. BDI Architecture (Preview)

The Beliefs-Desires-Intentions (BDI) model gives agents a formal cognitive structure inspired by human practical reasoning (philosopher Michael Bratman, 1987). We cover BDI in depth in Session 03 — here is a brief preview.

BDI Components

Beliefs: The agent's model of the world — what it thinks is true.
Desires: The agent's goals — what it wants to achieve.
Intentions: Committed plans the agent has chosen to pursue (a subset of desires).

The BDI interpreter loop continuously updates beliefs, selects desires to pursue, and commits to intention-achieving plans.

Choosing the Right Architecture

Use this decision guide when designing a new agent:

  • Real-time response required (<100ms)? → Reactive layer is mandatory.
  • Task requires multi-step planning? → Add a deliberative or BDI layer.
  • Environment changes unpredictably? → Hybrid architecture for resilience.
  • Goal can change mid-task? → BDI with intention revision.
  • Multiple concurrent goals? → BDI or multi-agent system.
LLM agents are implicitly hybrid: The LLM's next-token generation is reactive (it responds immediately to context), while chain-of-thought and tool-use planning add deliberative capability. Understanding this helps you debug why agents behave unexpectedly — often the reactive and deliberative "layers" are fighting each other.

Lab: Design an Architecture for a Customer Support Agent

You will sketch and implement a three-layer architecture for a customer support agent that handles refunds, escalations, and FAQs.

1

Define the Three Layers

# Three-layer architecture for a customer support agent

# Layer 1: REACTIVE — immediate classification
REACTIVE_RULES = {
    "urgent":   ["emergency", "broken", "not working", "critical"],
    "refund":   ["refund", "return", "money back", "cancel"],
    "billing":  ["invoice", "charge", "payment", "bill"],
    "general":  [],  # fallback
}

def reactive_layer(message: str) -> str:
    """Classify intent instantly from keywords."""
    msg_lower = message.lower()
    for intent, keywords in REACTIVE_RULES.items():
        if any(kw in msg_lower for kw in keywords):
            return intent
    return "general"

# Test
test_msgs = [
    "My service is completely broken and critical!",
    "I want a refund for order #1234",
    "What are your business hours?",
]
for msg in test_msgs:
    print(f"  '{msg[:40]}...' → {reactive_layer(msg)}")

Build the Executive Layer

# Layer 2: EXECUTIVE — sequences actions based on intent

def executive_layer(intent: str, customer_tier: str) -> list:
    """
    Returns an ordered list of actions to execute.
    The executive layer translates intent into an action plan.
    """
    plans = {
        "urgent": [
            "create_priority_ticket",
            "notify_on_call_engineer",
            "send_acknowledgement_email",
        ],
        "refund": [
            "lookup_order",
            "check_refund_eligibility",
            "process_refund" if customer_tier == "premium" else "escalate_to_human",
        ],
        "billing": [
            "lookup_billing_account",
            "generate_invoice_summary",
            "send_response",
        ],
        "general": [
            "search_knowledge_base",
            "send_response",
        ],
    }
    return plans.get(intent, plans["general"])

# Test
print("Urgent plan:", executive_layer("urgent", "standard"))
print("Refund plan (premium):", executive_layer("refund", "premium"))
print("Refund plan (standard):", executive_layer("refund", "standard"))
3

Add the Deliberative Layer

from datetime import datetime

# Layer 3: DELIBERATIVE — long-term goal management
class DeliberativeLayer:
    def __init__(self):
        self.active_goals = []
        self.world_state = {}

    def add_goal(self, goal: dict):
        self.active_goals.append(goal)
        print(f"  Goal added: {goal['name']}")

    def update_world_state(self, key: str, value):
        self.world_state[key] = value

    def select_next_goal(self) -> dict:
        """Simple priority-based goal selection."""
        if not self.active_goals:
            return None
        # Sort by priority (higher = more urgent)
        return sorted(self.active_goals, key=lambda g: g.get("priority", 0), reverse=True)[0]

# Assemble the full agent
class CustomerSupportAgent:
    def __init__(self):
        self.deliberative = DeliberativeLayer()

    def handle(self, message: str, customer_tier: str = "standard") -> dict:
        # Layer 1: Reactive classification
        intent = reactive_layer(message)
        # Layer 2: Executive planning
        action_plan = executive_layer(intent, customer_tier)
        # Layer 3: Deliberative goal tracking
        self.deliberative.add_goal({"name": f"resolve_{intent}", "priority": 3 if intent == "urgent" else 1})
        return {
            "intent": intent,
            "plan": action_plan,
            "active_goals": len(self.deliberative.active_goals),
            "timestamp": datetime.now().isoformat()
        }

agent = CustomerSupportAgent()
result = agent.handle("My payment isn't processing — this is critical!", "premium")
print("\nAgent response:", result)
