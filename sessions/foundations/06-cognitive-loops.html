The Sense-Think-Act Cycle

Every agent — from a thermostat to Claude Computer Use — operates on some variant of the Sense-Think-Act cycle. This is the most fundamental abstraction in agent design, predating AI by decades (it originated in cybernetics and control theory in the 1940s).

PhaseWhat happensIn an LLM agent
SenseCollect raw data from the environmentTool results, user messages, screenshots, API responses
ThinkProcess sensed data, update world model, decide actionLLM prompt + context window → next action token
ActExecute the chosen action, producing side effectsTool call, API request, file write, browser click

The OODA Loop

The OODA loop — Observe, Orient, Decide, Act — was developed by US Air Force Colonel John Boyd in the 1970s to describe fighter pilot decision-making. It adds a crucial step that raw Sense-Think-Act omits: Orient.

  • Observe: Gather raw sensory data (same as Sense).
  • Orient: Interpret data in context — apply prior knowledge, mental models, experience. This is where bias and expertise live.
  • Decide: Select a course of action from options generated during orientation.
  • Act: Execute the decision, then the loop restarts with new observations shaped by the action's effects.

The Orient step is what distinguishes an expert from a novice — both see the same data but interpret it differently. In LLM agents, orientation is performed by the system prompt + in-context examples + retrieved knowledge. This is why prompt engineering and RAG are so impactful: they improve the agent's orientation quality.

Boyd's key insight: Speed through the OODA loop matters — but only if orientation quality is maintained. An agent that acts fast but orients poorly will make fast wrong decisions. Better to slow down and orient correctly.

Environment Taxonomy

The type of environment your agent operates in fundamentally determines which architecture and algorithms are appropriate. Russell & Norvig (AIMA) classify environments along several dimensions:

DimensionOptionsAgent design implication
ObservabilityFully / Partially observablePartial → agent must maintain internal state (beliefs); Full → no internal state needed
DeterminismDeterministic / StochasticStochastic → agent must handle uncertainty, use probability estimates
EpisodicityEpisodic / SequentialSequential → past actions affect future; agent needs memory
DynamismStatic / DynamicDynamic → world changes while agent thinks; fast cognition loops required
ContinuityDiscrete / ContinuousContinuous action/state spaces require approximation strategies
AgentsSingle / Multi-agentMulti-agent → coordination, competition, and communication protocols needed

A web browsing agent operates in a partially observable, stochastic, sequential, dynamic, discrete, single-agent environment. This tells us it needs: a world model (beliefs), uncertainty handling, memory, fast loops, and robust error recovery.

Observation Preprocessing Pipelines

Raw observations are rarely ready for LLM consumption. A web page might be a 50,000-token HTML blob — far too large for a context window. Observation preprocessing transforms raw environment data into compact, relevant signals:

  • Truncation & chunking: Split long documents into chunks; pass only the most relevant.
  • Extraction: Pull structured data from HTML (BeautifulSoup), PDFs (pdfminer), or images (OCR).
  • Summarisation: Use a cheaper LLM to compress a long page into key facts before passing to the main agent.
  • Filtering: Remove navigation menus, ads, and boilerplate — keep only the content relevant to the current task.
  • Normalisation: Convert tool results to a uniform format (e.g., always return JSON with keys: content, source, timestamp).
from bs4 import BeautifulSoup
import re

def preprocess_webpage(html: str, max_chars: int = 4000) -> str:
    """Extract clean text from raw HTML for LLM consumption."""
    soup = BeautifulSoup(html, "html.parser")
    # Remove noise elements
    for tag in soup(["script", "style", "nav", "footer", "header", "aside"]):
        tag.decompose()
    # Extract main content
    main = soup.find("main") or soup.find("article") or soup.body
    text = main.get_text(separator="\n", strip=True) if main else soup.get_text()
    # Normalise whitespace
    text = re.sub(r'\n{3,}', '\n\n', text)
    # Truncate to context budget
    if len(text) > max_chars:
        text = text[:max_chars] + "\n... [truncated]"
    return text
The observation budget: Every token in your observation costs money and competes with reasoning tokens in the context window. Treat observation preprocessing as a first-class engineering concern, not an afterthought.

from dataclasses import dataclass, field
from typing import Optional
import time

@dataclass
class BrowserState:
    url: str = "about:blank"
    title: str = ""
    content: str = ""
    links: list = field(default_factory=list)
    last_action: str = ""
    timestamp: float = field(default_factory=time.time)

class SimulatedBrowser:
    """Simulates a browser for testing without real network calls."""
    PAGES = {
        "https://example.com": BrowserState(
            url="https://example.com",
            title="Example Domain",
            content="This domain is for use in illustrative examples in documents.",
            links=["https://example.com/about", "https://iana.org"]
        ),
        "https://example.com/about": BrowserState(
            url="https://example.com/about",
            title="About Example",
            content="Example.com is maintained by IANA. It was established in 1992.",
            links=["https://example.com"]
        ),
    }
    def navigate(self, url: str) -> BrowserState:
        state = self.PAGES.get(url, BrowserState(url=url, content="Page not found", title="404"))
        state.last_action = f"navigate({url})"
        return state

browser = SimulatedBrowser()
state = browser.navigate("https://example.com")
print("Initial state:", state.title)
print("Content:", state.content)

Implement the OODA Loop

def observe(browser_state: BrowserState) -> dict:
    """Sense: collect raw environment state."""
    return {
        "url": browser_state.url,
        "title": browser_state.title,
        "content": browser_state.content[:500],
        "links": browser_state.links[:5],
    }

def orient(observation: dict, goal: str, history: list) -> dict:
    """Orient: interpret observations in context of goal."""
    relevant_links = [l for l in observation["links"] if "about" in l.lower() or "info" in l.lower()]
    progress = "started" if not history else "in_progress"
    return {
        "goal": goal,
        "current_page": observation["title"],
        "goal_achieved": goal.lower() in observation["content"].lower(),
        "relevant_links": relevant_links,
        "steps_taken": len(history),
        "progress": progress,
    }

def decide(orientation: dict) -> dict:
    """Decide: choose next action based on oriented world model."""
    if orientation["goal_achieved"]:
        return {"action": "finish", "reason": "Goal found in current page content"}
    if orientation["relevant_links"]:
        return {"action": "navigate", "url": orientation["relevant_links"][0], "reason": "Found relevant link"}
    if orientation["steps_taken"] >= 5:
        return {"action": "finish", "reason": "Max steps reached, reporting best guess"}
    return {"action": "navigate", "url": "https://example.com/about", "reason": "Exploring available links"}

def act(decision: dict, browser: SimulatedBrowser) -> BrowserState:
    """Act: execute decision."""
    if decision["action"] == "navigate":
        return browser.navigate(decision["url"])
    return None  # finish

# Run the full OODA loop
goal = "When was example.com established?"
history = []
state = browser.navigate("https://example.com")

for cycle in range(5):
    print(f"\n=== OODA Cycle {cycle+1} ===")
    obs = observe(state)
    ori = orient(obs, goal, history)
    dec = decide(ori)
    print(f"  Observe: {obs['title']}")
    print(f"  Orient:  goal_achieved={ori['goal_achieved']}, relevant_links={ori['relevant_links']}")
    print(f"  Decide:  {dec['action']} — {dec['reason']}")
    if dec["action"] == "finish":
        print(f"  Act: DONE — {dec['reason']}")
        break
    new_state = act(dec, browser)
    history.append({"cycle": cycle, "decision": dec})
    state = new_state
    print(f"  Act: Navigated to {state.url}")

Add Observation Preprocessing

import re

def preprocess_observation(raw_content: str, goal: str, max_tokens: int = 500) -> str:
    """Filter and compress raw content to focus on goal-relevant info."""
    # Split into sentences
    sentences = re.split(r'(?<=[.!?])\s+', raw_content)
    # Score each sentence by goal keyword overlap
    goal_words = set(goal.lower().split())
    scored = []
    for sent in sentences:
        score = sum(1 for w in goal_words if w in sent.lower())
        scored.append((score, sent))
    # Sort by relevance, take top sentences within budget
    scored.sort(reverse=True)
    result, chars = [], 0
    for score, sent in scored:
        if chars + len(sent) > max_tokens * 4: break
        result.append(sent)
        chars += len(sent)
    return " ".join(result) if result else raw_content[:max_tokens*4]

# Test preprocessing
raw = "This domain is for use in examples. Example.com was established in 1992 by IANA. It serves as a placeholder domain."
processed = preprocess_observation(raw, goal="When was example.com established?")
print("Processed observation:", processed)
