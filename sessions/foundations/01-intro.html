Defining Agentic AI

The word "agentic" comes from agency — the capacity to act independently in pursuit of a goal. An agentic AI system is one that perceives its environment, reasons about what to do, takes actions (often using external tools), observes the outcomes, and loops back — all without a human directing each step.

This is a fundamentally different paradigm from the chatbot you might be familiar with. A chatbot waits for you to ask a question, generates one response, and stops. An agentic system is given a goal and pursues it across many steps, making decisions at each juncture.

Core definition: An AI agent is a system that: (1) perceives its environment, (2) maintains some form of state or memory, (3) selects and executes actions, and (4) does so in a goal-directed loop — autonomously, until the task is complete or impossible.

The Perception-Action Loop

At the heart of every agentic system is a cycle sometimes called the perception-action loop or the sense-think-act cycle. Here is how it works in detail:

  1. Perceive: The agent receives input — a user message, a tool result, an environment state, or a combination of all three.
  2. Represent: The agent updates its internal world model (its "beliefs") based on what it just perceived.
  3. Deliberate: The LLM backbone reasons about the current state, available tools, and goal to decide the next action.
  4. Act: The agent executes the chosen action — calling a tool, writing to memory, querying a database, or producing a final answer.
  5. Observe: The result of the action becomes a new perception, and the loop restarts.

This loop continues until the agent reaches a terminal state: either the goal is achieved, the agent decides it cannot proceed, or a maximum step count is reached.

Real-World Examples of Agentic AI

Agentic AI has moved from research papers to production systems. Here are four landmark systems that illustrate different points on the autonomy spectrum:

AutoGPT (2023)

One of the first open-source autonomous agents. Given a goal like "Research competitors and write a market report," AutoGPT would break the goal into sub-tasks, search the web, write notes to memory, and iterate — entirely without human prompting. It demonstrated the power and the pitfalls of fully autonomous agents (it often got stuck in loops or hallucinated data).

Devin (Cognition, 2024)

Devin is a software engineering agent that can read a GitHub issue, write code, run tests, debug failures, and open a pull request — all autonomously. It achieved a 13.86% solve rate on SWE-Bench, a benchmark of real-world GitHub issues. Devin lives near the "fully autonomous" end of the spectrum for coding tasks.

Claude Computer Use (Anthropic, 2024)

Claude's Computer Use capability allows the model to see a screenshot, move the cursor, click, type, and navigate applications. This is a perception-action loop applied to a computer desktop — the agent perceives the screen state, decides what to click, acts, and perceives the result. It is supervised-autonomous: Claude asks for confirmation on risky actions.

LangGraph / LangChain Agents (2024)

The LangGraph framework lets developers build agents as explicit state machines with typed states, conditional edges, and human-in-the-loop breakpoints. This gives fine-grained control over the autonomy level — you decide exactly which nodes require human approval.

Why Agentic AI Changes Everything

Chatbots augment individual tasks. Agents augment workflows. The economic implication is enormous: knowledge work that used to require a team of specialists can increasingly be orchestrated by an agent that calls specialist tools, reviews its own output, and iterates to quality.

Key insight for 2025 and beyond: The bottleneck is no longer model intelligence — it is agent reliability. The hard problems are: keeping agents on-task over long horizons, preventing them from causing unintended side-effects, and knowing when to pause and ask a human. These are the problems this course is designed to teach you to solve.

The Minimal Agent in Code

# The simplest possible agent loop — conceptual pseudocode
# This pattern underpins every framework: LangChain, LangGraph, AutoGen, CrewAI

def agent_loop(goal: str, tools: dict, memory: list, max_steps: int = 20):
    observation = goal
    for step in range(max_steps):
        # LLM reasons about what to do next
        decision = llm.reason(
            observation=observation,
            memory=memory,
            available_tools=list(tools.keys())
        )
        # Terminal condition
        if decision.action == "final_answer":
            return decision.answer
        # Execute the chosen tool
        tool_fn = tools[decision.action]
        result = tool_fn(**decision.arguments)
        # Update memory and loop
        memory.append({
            "step": step,
            "observation": observation,
            "action": decision.action,
            "result": result
        })
        observation = result   # result becomes next observation
    raise TimeoutError("Agent did not complete within max_steps")

Build a Classification Rubric

Create a scoring function based on four criteria: planning ability, tool use, memory persistence, and human oversight required.

def score_system(planning, tool_use, memory, human_oversight_low):
    """
    planning: 0-2 (0=none, 1=single-step, 2=multi-step)
    tool_use: 0-2 (0=none, 1=fixed tools, 2=dynamic tool selection)
    memory: 0-1 (0=stateless, 1=persistent memory)
    human_oversight_low: 0-1 (0=high oversight needed, 1=runs autonomously)
    Returns autonomy level 1-5
    """
    total = planning + tool_use + memory + human_oversight_low
    if total <= 1:   return 1
    elif total <= 2: return 2
    elif total <= 3: return 3
    elif total <= 5: return 4
    else:            return 5

# Example: classify a simple FAQ bot
faq_score = score_system(planning=0, tool_use=0, memory=0, human_oversight_low=0)
print(f"FAQ Bot → Level {faq_score}: {AUTONOMY_LEVELS[faq_score]}")
