Master the BDI cognitive model — the theoretical backbone of goal-directed autonomous agents.

Origins of the BDI Model

The BDI (Beliefs-Desires-Intentions) model was formalised by philosopher Michael Bratman in 1987 in his work Intention, Plans, and Practical Reason. Bratman was studying how humans manage competing goals and commit to plans despite limited time and resources. AI researchers Rao and Georgeff adapted this into a computational model in 1991, which became the foundation for the Procedural Reasoning System (PRS) — one of the first deployed multi-agent systems.

Why BDI matters for LLM agents: BDI gives us precise vocabulary to describe what agents "know," "want," and "plan to do" — vocabulary that maps directly to prompt design, memory architecture, and goal management in modern systems.

The Three Components

Beliefs (B)

The agent's model of the world — what it thinks is currently true. Beliefs can be incomplete or wrong. They are updated as new information arrives.

LLM analogy: system prompt + conversation history + retrieved context

Desires (D)

The agent's motivational states — all the things it would like to achieve. Multiple desires can exist simultaneously and may conflict with each other.

LLM analogy: user-provided goals + implicit objectives (be helpful, be safe)

Intentions (I)

The subset of desires the agent has committed to pursue. Intentions constrain future reasoning — the agent doesn't constantly re-evaluate committed plans.

LLM analogy: the current task plan selected by the planner node

The BDI Interpreter Loop

The BDI model is not just a data structure — it is an operational loop. At each cycle the interpreter:

  1. Update Beliefs: Process new perceptions and update the belief base.
  2. Generate Options (Desires): Based on current beliefs and active intentions, generate a set of possible desires.
  3. Filter (Select Intentions): Choose which desires to commit to as intentions, balancing feasibility and priority.
  4. Execute Intention: Select the next action from the current intention's plan and execute it.
  5. Check Completion: If the intention is achieved or impossible, drop it and return to step 2.

Intention Reconsideration

A subtle but critical aspect of BDI is when to reconsider intentions. Two extreme strategies:

StrategyBehaviourProblem
FanaticNever reconsiders — always completes the current intentionMisses opportunities when the world changes; pursues impossible goals
CautiousReconsiders after every actionToo much planning overhead; never commits long enough to succeed
BalancedReconsiders only when the world changes significantlyRequires good change-detection heuristics

Modern LLM agent frameworks implement "balanced" reconsideration through mechanisms like LangGraph's conditional edges — the agent checks whether its current plan is still valid at each node and can branch to a replanning node if needed.

The PRS System

The Procedural Reasoning System (PRS), built by Rao and Georgeff at SRI International, was one of the first full BDI implementations. It used a plan library — a database of pre-written plans (called KA plans) indexed by triggering conditions. When a desire matched a plan's condition, PRS would select that plan as an intention.

PRS was deployed in real applications including NASA's spacecraft monitoring systems and air traffic management. This demonstrated that BDI was not just theory — it could handle real-world complexity.

BDI Python Skeleton

from dataclasses import dataclass, field
from typing import Any, Callable, Optional
import time

@dataclass
class Belief:
    key: str
    value: Any
    confidence: float = 1.0  # 0.0 - 1.0
    timestamp: float = field(default_factory=time.time)

@dataclass
class Desire:
    name: str
    condition: Callable  # returns True when desire is satisfied
    priority: int = 1    # higher = more important

@dataclass
class Plan:
    name: str
    trigger: str         # desire name this plan achieves
    steps: list          # list of callables (actions)
    precondition: Callable = lambda beliefs: True

class BDIAgent:
    def __init__(self, name: str):
        self.name = name
        self.beliefs: dict[str, Belief] = {}
        self.desires: list[Desire] = []
        self.intentions: list[Plan] = []  # committed plans
        self.plan_library: list[Plan] = []

    def update_belief(self, key: str, value: Any, confidence: float = 1.0):
        self.beliefs[key] = Belief(key, value, confidence)
        print(f"[{self.name}] Belief updated: {key} = {value}")

    def add_desire(self, desire: Desire):
        self.desires.append(desire)

    def add_plan(self, plan: Plan):
        self.plan_library.append(plan)

    def select_intention(self) -> Optional[Plan]:
        """Filter desires → select highest-priority achievable desire → find matching plan."""
        achievable = [d for d in self.desires if not d.condition(self.beliefs)]
        if not achievable:
            return None
        best_desire = max(achievable, key=lambda d: d.priority)
        # Find a plan for this desire whose preconditions are met
        for plan in self.plan_library:
            if plan.trigger == best_desire.name and plan.precondition(self.beliefs):
                return plan
        return None

    def run_cycle(self):
        print(f"\n[{self.name}] --- BDI Cycle ---")
        intention = self.select_intention()
        if not intention:
            print(f"[{self.name}] No applicable intention found.")
            return False
        print(f"[{self.name}] Executing plan: {intention.name}")
        for step in intention.steps:
            step(self)
        return True


# Example usage
def pickup_package(agent):
    print(f"  Action: Moving to pickup location")
    agent.update_belief("has_package", True)

def deliver_package(agent):
    print(f"  Action: Delivering package to destination")
    agent.update_belief("delivered", True)
    agent.update_belief("has_package", False)

agent = BDIAgent("DeliveryBot")
agent.update_belief("has_package", False)
agent.update_belief("delivered", False)

agent.add_desire(Desire(
    name="deliver_item",
    condition=lambda b: b.get("delivered", Belief("delivered", False)).value,
    priority=5
))

agent.add_plan(Plan(
    name="pickup_and_deliver",
    trigger="deliver_item",
    steps=[pickup_package, deliver_package],
    precondition=lambda b: not b.get("has_package", Belief("hp", False)).value
))

agent.run_cycle()

from dataclasses import dataclass, field
from typing import Any, Callable, Optional, List
import random

@dataclass
class Belief:
    key: str
    value: Any

class BeliefBase:
    def __init__(self):
        self._beliefs = {}
    def set(self, key, value):
        self._beliefs[key] = value
        print(f"  [BELIEF] {key} = {value}")
    def get(self, key, default=None):
        return self._beliefs.get(key, default)
    def __repr__(self):
        return str(self._beliefs)

Run the BDI Interpreter Loop

def bdi_loop(bb, desires, plan_library, max_cycles=10):
    for cycle in range(max_cycles):
        print(f"\n=== Cycle {cycle+1} | Battery: {bb.get('battery')}% | Location: {bb.get('location')} ===")
        # Select highest-priority unsatisfied desire
        active = [d for d in desires if not d["satisfied"](bb)]
        if not active:
            print("All goals achieved!")
            return True
        intention = max(active, key=lambda d: d["priority"])
        print(f"  Intention: {intention['name']}")
        # Execute plan
        plan = plan_library.get(intention["name"], [])
        for action_name, action_fn in plan:
            print(f"  Executing: {action_name}")
            action_fn(bb)
        # Simulate battery drain
        bb.set("battery", max(0, bb.get("battery") - 10))
    print("Max cycles reached.")
    return False

bdi_loop(bb, desires, plan_library)

Extend: Add Intention Reconsideration

Modify the loop so that if the battery drops below 20% mid-execution, the current plan is abandoned and the "recharge" desire takes priority. Test by setting battery to 25% at the start.

def bdi_loop_with_reconsideration(bb, desires, plan_library, max_cycles=15):
    for cycle in range(max_cycles):
        print(f"\n=== Cycle {cycle+1} ===")
        active = [d for d in desires if not d["satisfied"](bb)]
        if not active:
            print("All goals achieved!")
            return
        intention = max(active, key=lambda d: d["priority"])
        print(f"  Intention: {intention['name']}")
        plan = plan_library.get(intention["name"], [])
        for action_name, action_fn in plan:
            # RECONSIDER before each step
            urgent = [d for d in desires if not d["satisfied"](bb) and d["priority"] > intention["priority"]]
            if urgent:
                print(f"  [RECONSIDER] Switching to: {urgent[0]['name']}")
                break
            print(f"  Executing: {action_name}")
            action_fn(bb)
            bb.set("battery", max(0, bb.get("battery") - 5))
