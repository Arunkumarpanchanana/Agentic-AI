Tool Design & API Integration

Build reliable, well-described tools that agents can select and use correctly every time.

35 min Capabilities Hands-on Lab 5 Quiz Questions

Tools Are the Agent's Hands

An LLM without tools is like a brilliant consultant locked in a room with no phone. Tools are what give agents the ability to affect the world — searching the web, reading files, calling APIs, running code, sending emails. But a poorly designed tool is worse than no tool: it confuses the agent, causes hallucinated arguments, and produces unreliable results.

Tool design is one of the highest-leverage skills in agentic AI engineering. A well-designed tool is reliable, self-describing, and error-tolerant. A poorly designed tool breaks agent workflows in subtle, hard-to-debug ways.

Tool anatomy: Every tool needs three things the LLM uses to decide whether and how to call it: (1) a name that clearly identifies what it does, (2) a description that explains when to use it and what it returns, and (3) a schema that defines its input parameters precisely.

Tool Anatomy

Error Handling Patterns

Tools will fail — APIs go down, rate limits are hit, queries return no results. The key is returning informative error messages that the agent can reason about, rather than raising exceptions that crash the loop.

import requests
from requests.exceptions import Timeout, ConnectionError

def robust_api_call(endpoint: str, params: dict) -> str:
    """Wrap an external API with robust error handling."""
    try:
        resp = requests.get(endpoint, params=params, timeout=10)
        resp.raise_for_status()
        data = resp.json()
        return format_results(data)
    except Timeout:
        return "ERROR: API request timed out after 10 seconds. Try again or use a different approach."
    except ConnectionError:
        return "ERROR: Cannot connect to the API. Check if the service is available."
    except requests.HTTPError as e:
        if e.response.status_code == 429:
            return "ERROR: Rate limit exceeded. Wait 60 seconds before retrying."
        elif e.response.status_code == 404:
            return "ERROR: Resource not found. Check if the query parameters are correct."
        return f"ERROR: HTTP {e.response.status_code}. {str(e)}"
    except Exception as e:
        return f"ERROR: Unexpected error: {str(e)[:200]}"

def format_results(data: dict) -> str:
    """Format API response as readable text for the LLM."""
    if not data.get("results"):
        return "No results found for this query. Try broadening your search terms."
    lines = [f"Found {len(data['results'])} results:"]
    for i, r in enumerate(data["results"][:5], 1):
        lines.append(f"{i}. {r.get('title', 'No title')} — {r.get('snippet', '')}")
    return "\n".join(lines)

Tool Selection Strategies

When an agent has many tools, selecting the right one becomes a challenge. Three strategies:

  • Description-based selection (default): The LLM reads all tool descriptions and selects based on semantic match. Works for up to ~20 tools.
  • Retrieval-based selection: For large tool libraries (100+), embed tool descriptions and retrieve the top-k most relevant tools before passing to the LLM.
  • Hierarchical routing: A lightweight classifier routes to a tool category; a specialist agent within that category selects the specific tool.
The description is part of your prompt: Tool descriptions consume context window tokens on every agent step. Keep them concise but complete — aim for 2-4 sentences per tool. Use negative examples ("do NOT use for X") to prevent common misuse.

Wrap the Open-Meteo Weather API

class WeatherInput(BaseModel):
    city: str = Field(description="City name, e.g. 'London', 'Tokyo'")
    days: int = Field(default=1, description="Number of forecast days (1-7)")

def get_weather(city: str, days: int = 1) -> str:
    """Get current weather and forecast for a city using Open-Meteo (free, no API key).
    Returns temperature, wind speed, and weather condition.
    Use for weather-related questions only."""
    try:
        # Geocoding
        geo = requests.get(f"https://geocoding-api.open-meteo.com/v1/search?name={city}&count=1", timeout=8)
        loc = geo.json()["results"][0]
        lat, lon = loc["latitude"], loc["longitude"]
        # Weather
        url = f"https://api.open-meteo.com/v1/forecast?latitude={lat}&longitude={lon}¤t_weather=true"
        w = requests.get(url, timeout=8).json()["current_weather"]
        return f"Weather in {city}: {w['temperature']}°C, wind {w['windspeed']} km/h, code {w['weathercode']}"
    except Exception as e:
        return f"ERROR fetching weather for {city}: {str(e)[:100]}"

weather_tool = StructuredTool.from_function(func=get_weather, name="get_weather",
    description=get_weather.__doc__, args_schema=WeatherInput)
print(weather_tool.run({"city": "Paris", "days": 1}))

Wrap a Wikipedia and Calculator Tool

class WikiInput(BaseModel):
    topic: str = Field(description="Topic to search on Wikipedia")

def search_wikipedia(topic: str) -> str:
    """Search Wikipedia for factual information about a topic.
    Returns a summary of the Wikipedia article.
    Use for historical facts, biographies, science concepts. NOT for real-time data."""
    try:
        url = f"https://en.wikipedia.org/api/rest_v1/page/summary/{topic.replace(' ','_')}"
        r = requests.get(url, timeout=8)
        if r.status_code == 404:
            return f"No Wikipedia article found for '{topic}'. Try a different search term."
        data = r.json()
        return f"{data['title']}: {data['extract'][:800]}"
    except Exception as e:
        return f"ERROR: {str(e)[:100]}"

class CalcInput(BaseModel):
    expression: str = Field(description="Python math expression, e.g. '2**10 / 3 + 5'")

def calculate(expression: str) -> str:
    """Evaluate a mathematical expression. Supports +,-,*,/,**,(), abs(), round().
    Returns the numeric result as a string. Use ONLY for math, not code execution."""
    safe_globals = {"__builtins__": {}, "abs": abs, "round": round, "pow": pow}
    try:
        result = eval(expression, safe_globals, {})
        return f"{expression} = {result}"
    except Exception as e:
        return f"ERROR evaluating '{expression}': {str(e)}"

wiki_tool = StructuredTool.from_function(func=search_wikipedia, name="search_wikipedia",
    description=search_wikipedia.__doc__, args_schema=WikiInput)
calc_tool = StructuredTool.from_function(func=calculate, name="calculator",
    description=calculate.__doc__, args_schema=CalcInput)
print(wiki_tool.run({"topic": "Eiffel Tower"}))
print(calc_tool.run({"expression": "1.7 ** 2 * 3.14159"}))

Wire Tools into a ReAct Agent

from langchain import hub
from langchain.agents import AgentExecutor, create_react_agent
from langchain_openai import ChatOpenAI

tools = [weather_tool, wiki_tool, calc_tool]
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = hub.pull("hwchase17/react")
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True, max_iterations=6)

result = executor.invoke({"input": "What is the weather in the city where the Eiffel Tower is located, and what is the square root of the tower's height in metres?"})
print("\nFinal answer:", result["output"])
