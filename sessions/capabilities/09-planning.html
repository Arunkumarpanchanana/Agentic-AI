Why Planning Is Hard

Planning — breaking a high-level goal into executable steps — is deceptively difficult. Humans do it implicitly using years of domain experience. Agents must do it explicitly, often in domains they've only seen during training. The challenge is that a naive plan can be technically correct but operationally useless: too abstract to execute, missing dependencies, or ignoring resource constraints.

Three planning approaches for LLM agents: Hierarchical Task Networks (top-down decomposition), Task Graphs (dependency-aware DAGs), and Tree of Thoughts (multi-path exploration with pruning).

Hierarchical Task Networks (HTN)

HTN planning decomposes a high-level goal into progressively more concrete subtasks through a hierarchy of methods. At the top is an abstract task; at the bottom are primitive actions that can be directly executed. Each method specifies how to decompose one abstract task into subtasks.

Abstract Task: "Write a research report on climate change"
  │
  ├─ Method 1: research-then-write
  │   ├─ Task: gather_sources(topic="climate change")
  │   │   ├─ search_web("climate change latest research")
  │   │   ├─ search_web("IPCC 2024 report")
  │   │   └─ fetch_arxiv_papers("climate change 2024")
  │   ├─ Task: synthesise_findings(sources)
  │   └─ Task: write_report(synthesis)
  │       ├─ write_introduction()
  │       ├─ write_body_sections()
  │       └─ write_conclusion()

Task Graphs and Dependency Resolution

A task graph (DAG) makes dependencies explicit. Tasks with no incoming edges can start immediately; tasks with incoming edges must wait for their dependencies to complete. This enables parallel execution of independent tasks.

from collections import defaultdict, deque

class TaskGraph:
    def __init__(self):
        self.tasks = {}       # id -> task dict
        self.deps = defaultdict(set)   # id -> set of dependency ids

    def add_task(self, task_id: str, description: str, dependencies: list = None):
        self.tasks[task_id] = {"id": task_id, "description": description, "status": "pending"}
        if dependencies:
            self.deps[task_id] = set(dependencies)

    def get_ready_tasks(self) -> list:
        """Return tasks whose dependencies are all completed."""
        completed = {tid for tid, t in self.tasks.items() if t["status"] == "done"}
        return [
            t for tid, t in self.tasks.items()
            if t["status"] == "pending" and self.deps[tid].issubset(completed)
        ]

    def mark_done(self, task_id: str):
        self.tasks[task_id]["status"] = "done"

    def topological_order(self) -> list:
        """Kahn's algorithm for topological sort."""
        in_degree = {tid: len(deps) for tid, deps in self.deps.items()}
        for tid in self.tasks:
            if tid not in in_degree:
                in_degree[tid] = 0
        queue = deque([tid for tid, deg in in_degree.items() if deg == 0])
        order = []
        while queue:
            node = queue.popleft()
            order.append(node)
            for tid, deps in self.deps.items():
                if node in deps:
                    in_degree[tid] -= 1
                    if in_degree[tid] == 0:
                        queue.append(tid)
        return order

# Example: Research report task graph
g = TaskGraph()
g.add_task("T1", "Search for climate data sources")
g.add_task("T2", "Search for policy papers")
g.add_task("T3", "Search for economic impact studies")
g.add_task("T4", "Synthesise all sources", dependencies=["T1","T2","T3"])
g.add_task("T5", "Write report draft", dependencies=["T4"])
g.add_task("T6", "Review and edit", dependencies=["T5"])

print("Execution order:", g.topological_order())
print("Ready to start:", [t["description"] for t in g.get_ready_tasks()])

Dynamic Replanning

Real tasks rarely go according to plan. A source might be unavailable, a search might return irrelevant results, or the agent discovers the goal was underspecified. Dynamic replanning handles this by invoking the planner again when a task fails:

def execute_with_dynamic_replan(graph: TaskGraph, executor_fn, planner_fn, goal: str):
    results = {}
    max_replans = 2
    replan_count = 0
    while True:
        ready = graph.get_ready_tasks()
        if not ready:
            all_done = all(t["status"] == "done" for t in graph.tasks.values())
            if all_done:
                print("All tasks complete!")
                return results
            else:
                print("Deadlock — no ready tasks but not all done")
                break
        for task in ready:
            try:
                result = executor_fn(task, results)
                results[task["id"]] = result
                graph.mark_done(task["id"])
            except Exception as e:
                if replan_count >= max_replans:
                    raise
                print(f"Task {task['id']} failed: {e}. Replanning...")
                new_tasks = planner_fn(goal, results, str(e))
                for t in new_tasks:
                    graph.add_task(**t)
                replan_count += 1
    return results

Tree of Thoughts

Tree of Thoughts (Yao et al., 2023) extends chain-of-thought by exploring multiple reasoning paths simultaneously and pruning unpromising branches. This is powerful for tasks where the optimal plan is not obvious upfront — like solving math problems or writing complex code.

  • Thought generation: At each step, generate k candidate thoughts (not just one).
  • Evaluation: Score each thought on likelihood of leading to the goal.
  • Search: Use BFS (explore all paths at each depth) or DFS with backtracking.
  • Pruning: Eliminate low-scoring paths to keep computation tractable.
Practical advice: Tree of Thoughts is expensive (k × depth LLM calls). Use it only for tasks where plan quality is far more important than cost — like generating a novel algorithm or solving a complex optimisation problem. For routine multi-step tasks, a simple DAG planner is sufficient.

from collections import defaultdict, deque

class TaskGraph:
    def __init__(self):
        self.tasks = {}
        self.deps = defaultdict(set)

    def add_task(self, task_id, description, dependencies=None):
        self.tasks[task_id] = {"id": task_id, "description": description, "status": "pending", "result": None}
        if dependencies:
            self.deps[task_id] = set(dependencies)

    def get_ready(self):
        done = {tid for tid, t in self.tasks.items() if t["status"] == "done"}
        return [t for tid, t in self.tasks.items() if t["status"] == "pending" and self.deps[tid].issubset(done)]

    def mark_done(self, tid, result=""):
        self.tasks[tid]["status"] = "done"
        self.tasks[tid]["result"] = result

    def summary(self):
        for tid, t in self.tasks.items():
            deps = list(self.deps.get(tid, []))
            print(f"  [{t['status'].upper():7}] {tid}: {t['description']} (deps: {deps})")

Simulate Parallel Execution

import time

def simulated_executor(task):
    """Fake task execution with variable duration."""
    durations = {"T1": 0.3, "T2": 0.4, "T3": 0.2, "T4": 0.5, "T5": 0.3, "T6": 0.6, "T7": 0.4, "T8": 0.2}
    time.sleep(durations.get(task["id"], 0.3))
    return f"Result of {task['description'][:40]}"

start = time.time()
wave = 0
while True:
    ready = g.get_ready()
    if not ready:
        if all(t["status"] == "done" for t in g.tasks.values()):
            print(f"\nAll tasks complete in {time.time()-start:.2f}s")
        else:
            print("Deadlock!")
        break
    wave += 1
    print(f"\nWave {wave} — running {len(ready)} tasks in parallel:")
    for task in ready:
        print(f"  Starting: {task['id']} — {task['description']}")
    for task in ready:
        result = simulated_executor(task)
        g.mark_done(task["id"], result)
        print(f"  ✓ Done: {task['id']}")

LLM-Powered Planner

from langchain_openai import ChatOpenAI
from langchain.schema import HumanMessage
import json

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

def llm_planner(goal: str) -> list:
    prompt = f"""Break this goal into a task DAG. Return JSON list only.
Each task: {{"id":"T1","description":"...","dependencies":[]}}
Use [] for tasks with no dependencies.

Goal: {goal}"""
    response = llm([HumanMessage(content=prompt)])
    tasks = json.loads(response.content)
    return tasks

goal = "Produce a competitive analysis report comparing Python, Go, and Rust for backend APIs"
tasks = llm_planner(goal)
print("LLM-generated plan:")
for t in tasks:
    print(f"  {t['id']}: {t['description']} → deps: {t['dependencies']}")

Visualise the DAG

def print_dag(tasks):
    """Print a simple ASCII representation of the task DAG."""
    dep_map = {t["id"]: t.get("dependencies", []) for t in tasks}
    roots = [t for t in tasks if not dep_map[t["id"]]]
    print("\nTask DAG:")
    printed = set()
    def print_tree(task_id, prefix="", tasks_by_id=None):
        if task_id in printed: return
        printed.add(task_id)
        desc = tasks_by_id[task_id]["description"][:50]
        print(f"{prefix}[{task_id}] {desc}")
        children = [t["id"] for t in tasks if task_id in dep_map[t["id"]]]
        for i, child in enumerate(children):
            is_last = i == len(children) - 1
            print_tree(child, prefix + ("    " if is_last else "│   "), tasks_by_id)

    tasks_by_id = {t["id"]: t for t in tasks}
    for root in roots:
        print_tree(root["id"], "", tasks_by_id)

print_dag(tasks)
