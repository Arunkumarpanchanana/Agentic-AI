Long-Horizon Task Execution

Build agents that persist across context window limits using checkpointing, state compression, and progress tracking.

35 min Capabilities Hands-on Lab 5 Quiz Questions

The Long-Horizon Problem

Most agentic tasks in tutorials complete in under 10 steps and a few minutes. Real-world tasks — conducting a literature review, building a production feature, auditing a codebase — may require hundreds of steps across hours or days. These long-horizon tasks expose fundamental limitations that short demos hide:

  • Context window exhaustion: Conversation history grows until it exceeds the model's context limit.
  • Goal drift: After many steps, the agent loses track of its original objective.
  • Cost explosion: Each step re-sends the full history, making costs grow quadratically.
  • No fault tolerance: A crash or API timeout at step 95 of 100 loses all progress.
Three pillars of long-horizon execution: Checkpointing (save/resume state), memory compression (keep context window manageable), and progress tracking (never lose sight of the goal).

Checkpointing Agent State

A checkpoint is a serialised snapshot of the agent's complete state at a point in time. If the agent crashes or is interrupted, it resumes from the last checkpoint rather than starting over.

import json, os, time
from dataclasses import dataclass, asdict, field
from typing import Any

@dataclass
class AgentCheckpoint:
    task_id: str
    goal: str
    step: int
    completed_tasks: list
    pending_tasks: list
    memory_summary: str
    results: dict
    timestamp: float = field(default_factory=time.time)

class CheckpointManager:
    def __init__(self, checkpoint_dir: str = "./checkpoints"):
        self.dir = checkpoint_dir
        os.makedirs(checkpoint_dir, exist_ok=True)

    def save(self, checkpoint: AgentCheckpoint):
        path = os.path.join(self.dir, f"{checkpoint.task_id}.json")
        with open(path, "w") as f:
            json.dump(asdict(checkpoint), f, indent=2)
        print(f"[CHECKPOINT] Saved at step {checkpoint.step} → {path}")

    def load(self, task_id: str) -> AgentCheckpoint | None:
        path = os.path.join(self.dir, f"{task_id}.json")
        if not os.path.exists(path):
            return None
        with open(path) as f:
            data = json.load(f)
        print(f"[CHECKPOINT] Loaded from step {data['step']}")
        return AgentCheckpoint(**data)

    def delete(self, task_id: str):
        path = os.path.join(self.dir, f"{task_id}.json")
        if os.path.exists(path):
            os.remove(path)

# Usage
manager = CheckpointManager()
cp = AgentCheckpoint(
    task_id="research-001",
    goal="Write a report on quantum computing",
    step=12,
    completed_tasks=["T1","T2","T3"],
    pending_tasks=["T4","T5"],
    memory_summary="Researched quantum gates and Shor's algorithm. Found 5 key papers.",
    results={"T1": "Quantum gates overview complete", "T2": "Shor's algorithm notes saved"}
)
manager.save(cp)
loaded = manager.load("research-001")
print(f"Resuming from step {loaded.step}, {len(loaded.pending_tasks)} tasks remaining")

Memory Compression

As the context window fills, older content must be compressed. Two strategies:

StrategyHow it worksWhen to use
Rolling summaryPeriodically summarise oldest N turns into a paragraph; keep only the summaryConversational agents, chat history
Key-fact extractionExtract named entities, decisions, and findings; discard proseResearch agents, structured tasks
 
from langchain_openai import ChatOpenAI
from langchain.schema import HumanMessage

compressor = ChatOpenAI(model="gpt-4o-mini", temperature=0)

def compress_history(history: list, max_items: int = 5) -> str:
    """Compress old history entries into a summary when history grows too long."""
    if len(history) <= max_items:
        return None  # No compression needed yet
    old_entries = history[:-max_items]
    old_text = "\n".join([f"Step {e['step']}: {e['action']} → {e['result'][:100]}" for e in old_entries])
    summary = compressor([HumanMessage(content=f"Summarise these agent steps into 3 key bullet points:\n{old_text}")]).content
    return summary

Progress Tracking

Goal drift — where the agent gradually loses track of the original objective — is one of the most subtle and common long-horizon failures. The solution is explicit progress tracking:

class ProgressTracker:
    def __init__(self, goal: str, total_tasks: int):
        self.goal = goal
        self.total = total_tasks
        self.done = 0
        self.milestones = []

    def update(self, task_name: str, result_summary: str):
        self.done += 1
        self.milestones.append({"task": task_name, "summary": result_summary})
        pct = (self.done / self.total) * 100
        print(f"[PROGRESS] {self.done}/{self.total} ({pct:.0f}%) — {task_name}")

    def goal_reminder(self) -> str:
        """Generate a goal-alignment reminder for the LLM context."""
        return f"""GOAL REMINDER: {self.goal}
Progress: {self.done}/{self.total} tasks complete.
Key milestones so far: {'; '.join([m['summary'][:80] for m in self.milestones[-3:]])}
Stay focused on the original goal."""
Checkpoint frequency: Save after every completed task (not every step). This balances fault tolerance against checkpoint overhead. For critical tasks with expensive steps, checkpoint before and after each irreversible action (sending an email, writing to a database, making an API call with side effects).

Build the Checkpoint System

import json, os, time
from dataclasses import dataclass, asdict, field

@dataclass
class Checkpoint:
    task_id: str; goal: str; step: int
    completed: list; pending: list; results: dict
    timestamp: float = field(default_factory=time.time)

def save_checkpoint(cp: Checkpoint, path="checkpoint.json"):
    with open(path, "w") as f: json.dump(asdict(cp), f, indent=2)
    print(f"  [SAVED] step={cp.step}, done={len(cp.completed)}/{len(cp.completed)+len(cp.pending)}")

def load_checkpoint(path="checkpoint.json"):
    if not os.path.exists(path): return None
    with open(path) as f: data = json.load(f)
    print(f"  [LOADED] Resuming from step {data['step']}")
    return Checkpoint(**data)

Build the Stateful Agent Loop

import random

def execute_task(task_name, step, fail_at=None):
    """Simulate task execution. Optionally fails at a specific step."""
    if step == fail_at:
        raise RuntimeError(f"Simulated failure at step {step}")
    time.sleep(0.1)
    return f"Result of {task_name} at step {step}"

def stateful_agent(goal, tasks, resume=True, fail_at=None):
    # Try to resume from checkpoint
    cp = load_checkpoint() if resume else None
    if cp and cp.goal == goal:
        completed = cp.completed
        pending = cp.pending
        results = cp.results
        step = cp.step
        print(f"Resuming from step {step}")
    else:
        completed, pending, results, step = [], list(tasks), {}, 0

    while pending:
        task = pending[0]
        step += 1
        print(f"  Step {step}: {task}")
        try:
            result = execute_task(task, step, fail_at=fail_at)
            results[task] = result
            completed.append(task)
            pending.pop(0)
            # Checkpoint after each task
            save_checkpoint(Checkpoint("r-001", goal, step, completed, pending, results))
        except Exception as e:
            print(f"  ERROR at step {step}: {e}")
            print("  State saved. Run again to resume.")
            return None

    print(f"\nAll {len(completed)} tasks complete!")
    os.remove("checkpoint.json")
    return results

tasks = [f"Task_{i}" for i in range(1,8)]
# First run — will fail at step 4
print("=== First run (will fail at step 4) ===")
stateful_agent("Research goal", tasks, resume=False, fail_at=4)

Add Memory Compression

def compress_results(results: dict, max_keep=3) -> str:
    """Compress old results into a summary string."""
    if len(results) <= max_keep:
        return str(results)
    items = list(results.items())
    old = items[:-max_keep]
    recent = dict(items[-max_keep:])
    summary = f"[COMPRESSED: {len(old)} earlier tasks completed] " + str(recent)
    print(f"  [COMPRESS] Reduced {len(results)} results to summary + {max_keep} recent")
    return summary

# Simulate growing results and compression
large_results = {f"T{i}": f"Result of task {i}" for i in range(1, 12)}
compressed = compress_results(large_results, max_keep=3)
print("Compressed state:", compressed[:200])
