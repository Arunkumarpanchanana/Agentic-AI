Memory Systems for Agents

Design working, episodic, semantic, and procedural memory for agents that remember across sessions.

35 min Capabilities Hands-on Lab 5 Quiz Questions

Why Agents Need Memory

A stateless LLM forgets everything after each API call. For short Q&A, that is fine. But agentic tasks — completing a week-long research project, maintaining a user relationship over months, building a codebase incrementally — require memory that persists beyond a single context window.

Memory enables agents to: avoid repeating work, learn from past mistakes, personalise responses to users, and maintain coherent state across many steps. Without it, every session starts cold.

Four memory types: Working (context window), Episodic (event log), Semantic (vector store of facts), Procedural (fine-tuned skills). Each has different read/write speeds, capacities, and use cases.

The Four Memory Types

1. Working Memory

The current context window — everything the LLM sees right now. Fast, expensive (costs tokens), ephemeral. Cleared at end of conversation.

Capacity: 8k–200k tokens | Speed: instant | Persistence: zero

2. Episodic Memory

A log of past events, interactions, and observations. Written sequentially, retrieved by time or similarity. Like a diary.

Capacity: unlimited | Speed: fast write, slow retrieval | Persistence: session or long-term

3. Semantic Memory

A knowledge base of facts, concepts, and relationships stored as vector embeddings. Retrieval by semantic similarity via cosine distance.

Capacity: millions of chunks | Speed: fast retrieval | Persistence: long-term

4. Procedural Memory

Skill memory — how to do things. Encoded in model weights via fine-tuning, or in tool definitions and prompt templates.

Capacity: model weights | Speed: instant (in weights) | Persistence: permanent until retrained

Memory Read/Write Patterns

PatternWhen to useImplementation
Write-on-eventSave important observations immediatelyStore tool results, key decisions to episodic log
Read-before-thinkRetrieve relevant context before reasoningVector similarity search on semantic store

Adding ChromaDB Memory to an Agent

import chromadb
from chromadb.utils import embedding_functions
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain.memory import VectorStoreRetrieverMemory
from langchain_community.vectorstores import Chroma

# --- Setup ChromaDB ---
embeddings = OpenAIEmbeddings()
vectorstore = Chroma(
    collection_name="agent_memory",
    embedding_function=embeddings,
    persist_directory="./agent_memory_db"
)

# --- Memory interface ---
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})
memory = VectorStoreRetrieverMemory(retriever=retriever)

# --- Write to memory ---
memory.save_context(
    inputs={"input": "What is the capital of France?"},
    outputs={"output": "The capital of France is Paris. Population ~2.1 million."}
)
memory.save_context(
    inputs={"input": "What language do they speak in France?"},
    outputs={"output": "French is the official language of France."}
)

# --- Read from memory ---
relevant = memory.load_memory_variables({"input": "Tell me about France"})
print("Retrieved memories:")
print(relevant["history"])
Memory architecture tip: Use a two-tier approach — fast in-memory cache (dict) for recent interactions, vector store for long-term retrieval. Most queries hit the cache; the vector store handles cross-session recall. This keeps latency low while maintaining depth.

Implement Memory Write and Read

from langchain_openai import OpenAIEmbeddings
import uuid, time

embeddings_model = OpenAIEmbeddings()

def remember(collection, user_msg, agent_response):
    """Write a conversation turn to vector memory."""
    text = f"User: {user_msg}\nAgent: {agent_response}"
    embedding = embeddings_model.embed_query(text)
    collection.add(
        ids=[str(uuid.uuid4())],
        embeddings=[embedding],
        documents=[text],
        metadatas=[{"timestamp": time.time(), "user": user_msg}]
    )
    print(f"  [MEMORY WRITE] Stored: '{user_msg[:50]}...'")

def recall(collection, query, n=3):
    """Retrieve most relevant past memories."""
    query_embedding = embeddings_model.embed_query(query)
    results = collection.query(query_embeddings=[query_embedding], n_results=n)
    return results["documents"][0] if results["documents"] else []

# Populate with some history
remember(collection, "I prefer concise answers", "Understood, I'll keep responses brief.")
remember(collection, "My name is Alex", "Nice to meet you, Alex!")
remember(collection, "I'm working on a Python project using FastAPI", "FastAPI is a great choice for async APIs.")
remember(collection, "What's a good ORM for FastAPI?", "SQLAlchemy with async support works well with FastAPI.")

# Test recall
memories = recall(collection, "What do you know about my project?")
print("\nRelevant memories retrieved:")
for m in memories:
    print(f"  - {m[:100]}")

Build a Memory-Augmented Agent

from langchain_openai import ChatOpenAI
from langchain.schema import HumanMessage, SystemMessage

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

def memory_agent(user_input: str, collection) -> str:
    # Retrieve relevant memories
    memories = recall(collection, user_input, n=3)
    memory_context = "\n".join(memories) if memories else "No relevant past interactions."
    # Build prompt with memory context
    messages = [
        SystemMessage(content=f"""You are a helpful assistant with memory of past conversations.
Relevant past interactions:
{memory_context}
Use this context to personalise your response."""),
        HumanMessage(content=user_input)
    ]
    response = llm(messages)
    answer = response.content
    # Save this interaction to memory
    remember(collection, user_input, answer)
    return answer

# Test the memory agent
print(memory_agent("Can you suggest a database for my project?", collection))
print()
print(memory_agent("What's my name again?", collection))

Test Memory Persistence

Switch ChromaDB to persistent mode and verify memory survives a Python restart:

import chromadb
# Use persistent client — data saved to disk
persistent_client = chromadb.PersistentClient(path="./my_agent_memory")
p_collection = persistent_client.get_or_create_collection("conversations")

# Write something
p_collection.add(ids=["test-1"], documents=["User prefers dark mode UI"], embeddings=[embeddings_model.embed_query("User prefers dark mode UI")])

# Simulate restart by creating a NEW client pointing to same path
client2 = chromadb.PersistentClient(path="./my_agent_memory")
c2 = client2.get_collection("conversations")
print("Records after restart:", c2.count())
results = c2.query(query_embeddings=[embeddings_model.embed_query("UI preferences")], n_results=1)
print("Retrieved:", results["documents"])
