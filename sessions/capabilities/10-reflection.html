Self-Reflection & Critique

Add Reflexion loops, inner monologue, and critic agents to dramatically improve output quality.

35 min Capabilities Hands-on Lab 5 Quiz Questions

The Reflexion Framework

Reflexion (Shinn et al., 2023) is a framework that allows agents to learn from failure through verbal reinforcement — instead of gradient-based learning, the agent reflects in natural language on what went wrong and commits that reflection to memory. On the next attempt, the reflection guides better decisions.

Reflexion outperformed GPT-4 (zero-shot) on HumanEval (coding), AlfWorld (decision-making), and HotpotQA (reasoning) by significant margins — without any fine-tuning, purely through iterative self-reflection.

Core loop: Actor generates a response → Evaluator scores it → Reflector writes a verbal critique → critique is stored in episodic memory → Actor uses critique on next attempt. Repeat until quality threshold is met or max trials reached.

The Three Roles in Reflexion

RoleResponsibilityImplementation
ActorGenerates the initial response or takes actionsStandard ReAct or LLM chain
EvaluatorScores the Actor's output against success criteriaLLM scorer, unit tests, or human feedback
ReflectorAnalyses failures and writes specific, actionable critiquesLLM with a critique prompt

Inner Monologue

Inner monologue is a lighter-weight form of self-reflection where the agent narrates its reasoning process to itself before committing to an answer. Unlike Reflexion (which spans multiple attempts), inner monologue happens within a single response. It is the "think out loud" pattern:

INNER_MONOLOGUE_PROMPT = """Before answering, think through the problem step by step:
1. What exactly is being asked?
2. What do I know that is relevant?
3. What could go wrong with a naive answer?
4. What is the most accurate, complete answer?

Then provide your final answer after your reasoning.

Question: {question}"""

This simple pattern significantly reduces reasoning errors on complex questions because it forces the model to identify its own uncertainty before answering.

Self-Evaluation Prompts

Effective self-evaluation prompts are specific and criterion-based. Generic "was this good?" prompts produce vague critiques that don't improve the next attempt:

QualityGeneric (bad)Specific (good)
Accuracy"Was this accurate?""List any factual claims that might be incorrect. For each, identify what would need to be verified."
Completeness"Did I cover everything?""What aspects of the question were not addressed? What would a domain expert expect to see that is missing?"
Clarity"Was this clear?""Which sentences would confuse a non-expert reader? How could they be rephrased?"
Action"How can I improve?""Give 3 concrete changes to make in the next draft, in order of impact."

The Critic Agent Pattern

A critic agent is a separate LLM instance (or same LLM with a different system prompt) whose sole job is to evaluate and critique the primary agent's output. Using a separate instance avoids the "self-serving bias" where an agent rates its own output too highly.

from langchain_openai import ChatOpenAI
from langchain.schema import SystemMessage, HumanMessage

actor_llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)
critic_llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

CRITIC_SYSTEM = """You are a rigorous critic. Your job is to identify weaknesses in a draft response.
Be specific. List exact issues, not vague complaints.
Rate overall quality 1-10 and provide exactly 3 actionable improvements."""

def reflexion_loop(question: str, max_trials: int = 3) -> str:
    reflections = []
    for trial in range(max_trials):
        # Actor: generate response with memory of past reflections
        reflection_context = "\n".join([f"Past attempt {i+1} critique: {r}" for i, r in enumerate(reflections)])
        actor_prompt = f"""Answer this question as accurately and completely as possible.
{f'Previous attempt critiques to address:{chr(10)}{reflection_context}' if reflections else ''}

Question: {question}"""
        response = actor_llm([HumanMessage(content=actor_prompt)]).content

        # Critic: evaluate the response
        critique = critic_llm([
            SystemMessage(content=CRITIC_SYSTEM),
            HumanMessage(content=f"Question: {question}\n\nDraft response:\n{response}\n\nYour critique:")
        ]).content

        print(f"\n--- Trial {trial+1} ---")
        print(f"Response (first 200 chars): {response[:200]}...")
        print(f"Critique: {critique[:300]}...")

        # Check if quality is sufficient (look for high score in critique)
        if "9/" in critique or "10/" in critique or "excellent" in critique.lower():
            print(f"\nQuality threshold met at trial {trial+1}")
            return response

        reflections.append(critique)

    print(f"\nMax trials reached. Returning best response.")
    return response

result = reflexion_loop("Explain the CAP theorem in distributed systems")
Cost management: Reflexion multiplies your LLM calls by the number of trials. Use it selectively — only for high-stakes outputs like final reports, generated code, or user-facing content. For intermediate reasoning steps, inner monologue is cheaper and often sufficient.

def critique_essay(essay, topic):
    return critic([
        SystemMessage(content="You are a writing coach. Be specific and constructive."),
        HumanMessage(content=f"""Critique this essay on '{topic}'.

{essay}""")
    ]).content
