Multi-Agent Coordination Theory

Game theory, cooperation vs competition, centralized vs decentralized control, and emergent coordination.

35 min Multi-Agent Hands-on Lab 5 Quiz Questions

Why Multiple Agents?

A single agent is limited by its context window, computational budget, and the breadth of skills one LLM can reliably deploy simultaneously. Multi-agent systems break these limits by distributing work across specialized agents that collaborate, each bringing focused capability to a shared task.

Multi-agent systems enable: parallel execution of independent subtasks, specialization (a code-writing agent, a testing agent, a documentation agent), redundancy through multiple independent checks, and scalability beyond single-context limits.

Key tension: More agents mean more coordination overhead. The art is finding the right number of agents and the right coordination mechanism for each task — more is not always better.

Game Theory Basics for Agent Designers

Game theory gives us precise language for reasoning about multi-agent interactions. Three foundational concepts:

The Prisoner's Dilemma

Two agents each choose to cooperate or defect. Mutual cooperation yields the best joint outcome, but each agent is individually tempted to defect (getting a higher payoff if the other cooperates). This models situations where agents have misaligned incentives despite benefiting from coordination.

In LLM agents: Two agents sharing a compute budget may each try to use more tokens, reducing quality for the other. Solution: explicit resource allocation rules.

Coordination Games

Agents need to make consistent choices but have multiple equilibria. The challenge: which one? Example: two agents both must use the same data format. Either JSON or XML works, but they must agree. Solution: a shared protocol (Session 14 covers this).

Cooperation vs Competition

Most LLM multi-agent systems are cooperative — agents share a goal and coordinate. But some systems deliberately introduce competition (e.g., debate agents, adversarial review) to improve output quality through productive conflict.

Centralized vs Decentralized Coordination

DimensionCentralizedDecentralized
ControlSingle orchestrator assigns tasksAgents negotiate directly
Single point of failureYes — orchestrator crash stops allNo — agents continue independently

PredictabilityHigh — orchestrator controls flowLower — emergent behaviour possible

Emergent Coordination

In complex multi-agent systems, coordination patterns can emerge without being explicitly programmed. This is borrowed from complexity science: simple local rules between agents produce complex global behaviours.

Example: In a document processing pipeline, if each agent marks completed documents with a "done" tag, a downstream agent can autonomously discover which documents to process next — without any central scheduler. The coordination emerges from the shared state.

import random
from dataclasses import dataclass, field
from typing import List

@dataclass
class Agent:
    agent_id: str
    specialty: str
    completed: list = field(default_factory=list)

    def can_handle(self, task: dict) -> bool:
        return task.get("type") == self.specialty

    def process(self, task: dict) -> dict:
        result = f"{self.agent_id} processed {task['name']}"
        self.completed.append(task["id"])
        return {"task_id": task["id"], "result": result, "agent": self.agent_id, "status": "done"}

class DecentralizedSystem:
    def __init__(self, agents: List[Agent]):
        self.agents = agents
        self.task_queue = []
        self.results = []

    def add_task(self, task: dict):
        self.task_queue.append({**task, "status": "pending"})

    def run_round(self):
        """Each agent picks tasks it can handle — emergent coordination."""
        pending = [t for t in self.task_queue if t["status"] == "pending"]
        for task in pending:
            eligible = [a for a in self.agents if a.can_handle(task)]
            if eligible:
                chosen = random.choice(eligible)  # random selection among eligible
                task["status"] = "taken"
                result = chosen.process(task)
                self.results.append(result)
                print(f"  {chosen.agent_id} → {task['name']}")

# Simulate
agents = [
    Agent("ResearchBot-1", "research"),
    Agent("ResearchBot-2", "research"),
    Agent("WriterBot-1",   "writing"),
    Agent("ReviewBot-1",   "review"),
]
system = DecentralizedSystem(agents)
tasks = [
    {"id":"t1","name":"Research quantum computing","type":"research"},
    {"id":"t2","name":"Research machine learning","type":"research"},
    {"id":"t3","name":"Write introduction","type":"writing"},
    {"id":"t4","name":"Review draft","type":"review"},
]
for t in tasks: system.add_task(t)
print("Running decentralized round:")
system.run_round()
print(f"\nCompleted {len(system.results)}/{len(tasks)} tasks")
Start centralized: When building multi-agent systems, start with a centralized orchestrator. It is easier to debug and reason about. Only move to decentralized coordination when you have a specific need (fault tolerance, scale, dynamic task discovery) that centralization cannot meet.

Define the Game

import random

# Shared resource: 100 tokens per round
TOTAL_TOKENS = 100
ROUNDS = 10

def payoff(my_request, other_request, total=TOTAL_TOKENS):
    """Calculate payoff given two agents' resource requests."""
    total_requested = my_request + other_request
    if total_requested <= total:
        return my_request  # Both get what they asked for
    else:
        # Proportional split when over-requested
        return int(total * my_request / total_requested)

# Test payoff matrix
print("Payoff matrix (my_request vs other_request):")
print(f"{'':>8}", end="")

Implement Agent Strategies

class ResourceAgent:
    def __init__(self, name, strategy):
        self.name = name
        self.strategy = strategy  # "greedy", "fair", "adaptive", "tit_for_tat"
        self.history = []
        self.total_payoff = 0

    def request(self, round_num) -> int:
        if self.strategy == "greedy":
            return 80  # Always ask for 80%
        elif self.strategy == "fair":
            return 50  # Always ask for half
        elif self.strategy == "adaptive":
            # Ask for more if last round gave full payoff
            if self.history and self.history[-1]["got"] == self.history[-1]["asked"]:
                return min(80, self.history[-1]["asked"] + 10)
            return 50
        elif self.strategy == "tit_for_tat":
            # Mirror the other agent's last request
            if not self.history: return 50
            return self.history[-1].get("other_asked", 50)
        return 50

    def update(self, asked, got, other_asked):
        self.history.append({"asked": asked, "got": got, "other_asked": other_asked})
        self.total_payoff += got

def run_simulation(strat_a, strat_b, rounds=ROUNDS):
    a = ResourceAgent("Agent A", strat_a)
    b = ResourceAgent("Agent B", strat_b)
    print(f"\n{'Strategy':>15}: {strat_a} vs {strat_b}")
    print(f"{'Round':>5} | {'A asks':>7} | {'B asks':>7} | {'A gets':>7} | {'B gets':>7}")
    print("-" * 45)
    for r in range(rounds):
        ra, rb = a.request(r), b.request(r)
        ga, gb = payoff(ra, rb), payoff(rb, ra)
        a.update(ra, ga, rb); b.update(rb, gb, ra)
        print(f"{r+1:>5} | {ra:>7} | {rb:>7} | {ga:>7} | {gb:>7}")
    print(f"Total: A={a.total_payoff}, B={b.total_payoff}")
    return a.total_payoff, b.total_payoff

run_simulation("fair", "fair")
run_simulation("greedy", "fair")
run_simulation("tit_for_tat", "greedy")

Add a Coordinator

def coordinated_allocation(requests: dict, total=TOTAL_TOKENS) -> dict:
    """Centralized coordinator enforces fair allocation."""
    total_requested = sum(requests.values())
    if total_requested <= total:
        return requests  # Everyone gets what they want
    # Proportional allocation with cap
    scale = total / total_requested
    return {agent: int(req * scale) for agent, req in requests.items()}

# Compare uncoordinated vs coordinated
agents_requests = {"Agent A": 80, "Agent B": 70, "Agent C": 60}
print("Requests:", agents_requests)
print("Without coordinator — first-come-first-served:")
remaining = TOTAL_TOKENS
for agent, req in agents_requests.items():
    got = min(req, remaining)
    remaining -= got
    print(f"  {agent}: asked {req}, got {got}")

print("\nWith coordinator — proportional allocation:")
allocated = coordinated_allocation(agents_requests)
for agent, tokens in allocated.items():
    print(f"  {agent}: asked {agents_requests[agent]}, got {tokens}")
