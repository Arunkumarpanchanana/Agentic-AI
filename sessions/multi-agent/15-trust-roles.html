Roles in Multi-Agent Systems

Roles define what an agent is responsible for and what it is permitted to do. Well-defined roles prevent capability overlap, reduce coordination confusion, and make the system easier to reason about and debug. Common role patterns:

RoleResponsibilitiesTypical permissions
OrchestratorAssigns tasks, monitors progress, resolves conflictsRead all state, assign any task, terminate agents
Specialist WorkerExecutes specific task types (research, coding, writing)Use designated tools, write to own result slot
Critic/ReviewerEvaluates outputs, provides feedbackRead all results, write critique; cannot modify originals
MonitorTracks system health, cost, timeoutsRead all state, raise alerts, can pause agents
GatewayManages external I/O (user, APIs, storage)External network access; other agents cannot access external directly
Principle of least privilege: Each agent should have only the permissions it needs for its role. A research agent should not be able to send emails; a writing agent should not be able to delete files. This limits blast radius when an agent misbehaves.

Capability Discovery

In dynamic multi-agent systems, the orchestrator may not know upfront which agents are available and what they can do. Capability discovery solves this through a registry where agents advertise their capabilities:

from dataclasses import dataclass, field
from typing import List, Dict

@dataclass
class AgentCapability:
    agent_id: str
    role: str
    skills: List[str]          # e.g., ["web_search", "summarization"]
    tools: List[str]           # e.g., ["duckduckgo", "calculator"]
    max_concurrent_tasks: int = 1
    trust_level: float = 0.5   # 0.0 (untrusted) to 1.0 (fully trusted)
    success_rate: float = 1.0  # computed from history

class AgentRegistry:
    def __init__(self):
        self._agents: Dict[str, AgentCapability] = {}

    def register(self, cap: AgentCapability):
        self._agents[cap.agent_id] = cap
        print(f"  [REGISTRY] Registered: {cap.agent_id} ({cap.role}) — skills: {cap.skills}")

    def find_capable(self, required_skill: str, min_trust: float = 0.5) -> List[AgentCapability]:
        """Find agents with required skill above trust threshold."""
        return [
            a for a in self._agents.values()
            if required_skill in a.skills and a.trust_level >= min_trust
        ]

    def best_for(self, skill: str) -> AgentCapability | None:
        candidates = self.find_capable(skill)
        if not candidates: return None
        return max(candidates, key=lambda a: a.trust_level * a.success_rate)

registry = AgentRegistry()
registry.register(AgentCapability("researcher-1", "specialist", ["web_search","summarization"], ["duckduckgo"], trust_level=0.9))
registry.register(AgentCapability("researcher-2", "specialist", ["web_search","data_extraction"], ["duckduckgo","scraper"], trust_level=0.7))
registry.register(AgentCapability("writer-1",     "specialist", ["writing","editing"], [], trust_level=0.85))

best = registry.best_for("web_search")
print(f"\nBest for web_search: {best.agent_id} (trust={best.trust_level})")

Trust Scoring Models

Trust scores quantify how reliable an agent is, based on its history of task completion. A simple trust model:

class TrustManager:
    def __init__(self, decay: float = 0.1):
        self.scores: Dict[str, float] = {}
        self.history: Dict[str, list] = {}
        self.decay = decay  # Recent performance weighted more

    def record(self, agent_id: str, success: bool, task_difficulty: float = 1.0):
        if agent_id not in self.scores:
            self.scores[agent_id] = 0.5  # Start neutral
        outcome = 1.0 if success else 0.0
        # Exponential moving average
        self.scores[agent_id] = (1 - self.decay) * self.scores[agent_id] + self.decay * outcome
        self.history.setdefault(agent_id, []).append({"success": success, "difficulty": task_difficulty})

    def get_trust(self, agent_id: str) -> float:
        return round(self.scores.get(agent_id, 0.5), 3)

trust = TrustManager()
# Simulate agent history
for _ in range(8): trust.record("researcher-1", success=True)
trust.record("researcher-1", success=False)  # One failure
for _ in range(3): trust.record("researcher-2", success=True)
trust.record("researcher-2", success=False)
trust.record("researcher-2", success=False)

print(f"researcher-1 trust: {trust.get_trust('researcher-1')}")
print(f"researcher-2 trust: {trust.get_trust('researcher-2')}")

The Principal-Agent Problem

The principal-agent problem from economics describes the tension when one entity (principal) delegates work to another (agent) whose interests may not perfectly align. In AI systems, this appears when:

  • A worker agent optimises for completing its assigned metric rather than the true goal.
  • An agent takes shortcuts that look like success but aren't (reward hacking).
  • An agent accumulates capabilities or resources beyond what the task requires.

Solutions: clear success criteria, output verification by a critic agent, permission boundaries, and audit logs of all agent actions.

Delegation with verification: Never fully trust a worker agent's self-reported success. Always have the orchestrator (or a dedicated verifier agent) independently check the output against the task specification before marking a task complete.

from dataclasses import dataclass, field
from typing import List, Dict
import random, time

@dataclass
class WorkerAgent:
    agent_id: str
    skills: List[str]
    reliability: float  # 0-1: probability of producing correct output
    trust_score: float = 0.5

    def execute(self, task: dict) -> dict:
        """Execute a task. Returns result (may be wrong based on reliability)."""
        time.sleep(0.05)
        success = random.random() < self.reliability
        return {
            "task_id": task["id"],
            "agent": self.agent_id,
            "result": f"Completed: {task['description']}" if success else f"WRONG: Partial result for {task['description']}",
            "self_reported_success": True,  # Agents always claim success!
        }

workers = [
    WorkerAgent("w1", ["research"], reliability=0.9),
    WorkerAgent("w2", ["research"], reliability=0.6),
    WorkerAgent("w3", ["writing"],  reliability=0.85),
]
print("Workers:", [(w.agent_id, w.skills, w.reliability) for w in workers])

Build the Manager with Trust Scoring

class ManagerAgent:
    def __init__(self, workers: List[WorkerAgent]):
        self.workers = {w.agent_id: w for w in workers}
        self.trust = {w.agent_id: 0.5 for w in workers}
        self.history = []

    def select_worker(self, skill: str) -> WorkerAgent:
        """Select highest-trust worker with required skill."""
        eligible = [w for w in self.workers.values() if skill in w.skills]
        if not eligible: raise ValueError(f"No worker for skill: {skill}")
        return max(eligible, key=lambda w: self.trust[w.agent_id])

    def verify(self, result: dict, task: dict) -> bool:
        """Independent verification: check result meets basic criteria."""
        r = result.get("result", "")
        return len(r) > 20 and "WRONG" not in r and task["description"][:10].lower() in r.lower()

    def update_trust(self, agent_id: str, verified_success: bool):
        """Update trust score based on verified (not self-reported) success."""
        decay = 0.15
        self.trust[agent_id] = (1-decay)*self.trust[agent_id] + decay*(1.0 if verified_success else 0.0)
        self.trust[agent_id] = round(self.trust[agent_id], 3)

    def delegate(self, task: dict) -> dict:
        worker = self.select_worker(task["skill"])
        print(f"  Delegating '{task['description'][:30]}' → {worker.agent_id} (trust={self.trust[worker.agent_id]:.2f})")
        result = worker.execute(task)
        verified = self.verify(result, task)
        self.update_trust(worker.agent_id, verified)
        print(f"  Self-reported: {result['self_reported_success']} | Verified: {verified} | New trust: {self.trust[worker.agent_id]:.2f}")
        self.history.append({"task": task["id"], "agent": worker.agent_id, "verified": verified})
        return {**result, "verified": verified}

manager = ManagerAgent(workers)
tasks = [
    {"id": f"t{i}", "description": f"Research topic {i} thoroughly", "skill": "research"}
    for i in range(1, 7)
]
print("\n=== Running delegated tasks ===")
for t in tasks:
    manager.delegate(t)

Add Automatic Blacklisting

BLACKLIST_THRESHOLD = 0.3

def delegate_with_blacklist(manager, task):
    worker = manager.select_worker(task["skill"])
    if manager.trust[worker.agent_id] < BLACKLIST_THRESHOLD:
        print(f"  ⛔ {worker.agent_id} blacklisted (trust={manager.trust[worker.agent_id]:.2f}). No eligible workers!")
        return None
    return manager.delegate(task)

# Simulate a bad worker dominating then getting blacklisted
bad_worker = WorkerAgent("w_bad", ["research"], reliability=0.1)
manager.workers["w_bad"] = bad_worker
manager.trust["w_bad"] = 0.5

bad_tasks = [{"id": f"b{i}", "description": f"Research bad topic {i}", "skill": "research"} for i in range(5)]
print("=== Bad worker scenario ===")
for t in bad_tasks:
    result = delegate_with_blacklist(manager, t)
    if not result: break
