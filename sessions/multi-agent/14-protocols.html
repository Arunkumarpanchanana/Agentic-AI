Communication Protocols

Master message passing, shared blackboard, pub/sub, and FIPA ACL speech acts for agent-to-agent communication.

Why Communication Protocols Matter

When multiple agents need to collaborate, they must agree on how to communicate — not just what to say. Without a protocol, agents produce incompatible message formats, misinterpret each other's intent, and create coordination failures that are extremely difficult to debug.

A communication protocol defines: message structure, the set of valid message types (performatives), how agents identify themselves and their target, and the expected conversation flow (who replies to whom in what order).

Four protocol patterns: Direct message passing (point-to-point), shared blackboard (shared state), pub/sub (topic-based broadcasting), and FIPA ACL (formal speech acts). Each suits different coordination needs.

1. Direct Message Passing

The simplest protocol: Agent A sends a structured message directly to Agent B. Works well for simple pipelines where the communication graph is known upfront.

from dataclasses import dataclass, field
from typing import Any
import time, uuid

@dataclass
class Message:
    sender: str
    receiver: str
    performative: str   # "inform", "request", "propose", "agree", "refuse"
    content: Any
    conversation_id: str = field(default_factory=lambda: str(uuid.uuid4())[:8])
    timestamp: float = field(default_factory=time.time)

    def __repr__(self):
        return f"[{self.performative.upper()}] {self.sender}→{self.receiver}: {str(self.content)[:60]}"

class MessageBus:
    """Simple in-process message bus for agent communication."""
    def __init__(self):
        self.inboxes: dict[str, list] = {}

    def register(self, agent_id: str):
        self.inboxes[agent_id] = []

    def send(self, msg: Message):
        if msg.receiver not in self.inboxes:
            raise ValueError(f"Unknown agent: {msg.receiver}")
        self.inboxes[msg.receiver].append(msg)
        print(f"  📨 {msg}")

    def receive(self, agent_id: str) -> list:
        msgs = self.inboxes.get(agent_id, [])
        self.inboxes[agent_id] = []
        return msgs

bus = MessageBus()
bus.register("planner"); bus.register("researcher"); bus.register("writer")

bus.send(Message("planner", "researcher", "request", {"task": "Research quantum computing trends 2024"}))
bus.send(Message("planner", "writer",     "request", {"task": "Draft introduction section"}))

msgs = bus.receive("researcher")
print(f"\nResearcher inbox: {len(msgs)} messages")

2. Shared Blackboard Architecture

A blackboard is a shared data structure that all agents can read from and write to. Agents don't communicate directly — they post results to the blackboard and read others' results. This decouples agents from each other, enabling flexible, emergent coordination.

import threading
from typing import Callable

class Blackboard:
    """Thread-safe shared state for multi-agent coordination."""
    def __init__(self):
        self._data: dict = {}
        self._lock = threading.Lock()
        self._watchers: dict[str, list[Callable]] = {}

    def write(self, key: str, value: Any, author: str = "unknown"):
        with self._lock:
            self._data[key] = {"value": value, "author": author, "timestamp": time.time()}
            print(f"  [BB WRITE] {author} → {key}: {str(value)[:50]}")
        # Notify watchers
        for watcher in self._watchers.get(key, []):
            watcher(key, value)

    def read(self, key: str) -> Any:
        with self._lock:
            entry = self._data.get(key)
            return entry["value"] if entry else None

    def watch(self, key: str, callback: Callable):
        """Register a callback triggered when key is written."""
        self._watchers.setdefault(key, []).append(callback)

bb = Blackboard()
# Researcher writes findings
bb.write("research_findings", "Quantum computing advances: IBM 1000-qubit chip...", "researcher")
# Writer reads findings
findings = bb.read("research_findings")
print(f"\nWriter read: {findings[:60] if findings else 'Nothing yet'}...")

3. Pub/Sub Pattern

In pub/sub, agents publish messages to named topics and subscribe to topics they care about. Publishers don't know who is listening; subscribers don't know who published. This is ideal for loosely coupled systems where agents may join or leave dynamically.

class PubSubBus:
    def __init__(self):
        self.subscribers: dict[str, list] = {}

    def subscribe(self, topic: str, agent_id: str, callback: Callable):
        self.subscribers.setdefault(topic, []).append((agent_id, callback))
        print(f"  [SUB] {agent_id} subscribed to '{topic}'")

    def publish(self, topic: str, message: Any, publisher: str):
        print(f"  [PUB] {publisher} → topic:'{topic}': {str(message)[:50]}")
        for agent_id, callback in self.subscribers.get(topic, []):
            callback(agent_id, topic, message)

pubsub = PubSubBus()
def handle_research(agent_id, topic, msg): print(f"    {agent_id} received on {topic}: {str(msg)[:40]}")
def handle_error(agent_id, topic, msg):    print(f"    {agent_id} ERROR ALERT on {topic}: {str(msg)[:40]}")

pubsub.subscribe("research_complete", "writer_agent",   handle_research)
pubsub.subscribe("research_complete", "review_agent",   handle_research)
pubsub.subscribe("error",             "monitor_agent",  handle_error)
pubsub.publish("research_complete", {"topic": "AI", "summary": "Key findings..."}, "researcher_agent")
pubsub.publish("error", {"code": 500, "msg": "API timeout"}, "researcher_agent")

4. FIPA ACL Speech Acts

FIPA (Foundation for Intelligent Physical Agents) ACL defines a formal set of performatives — message types that indicate the communicative intent:

PerformativeMeaningExample
informAssert that something is true"The search returned 42 results."
requestAsk another agent to perform an action"Please search for X and return results."
proposeOffer to do something under stated conditions"I can complete this in 2 steps if given tool access."
agreeAccept a previous proposal or request"I agree to handle the research subtask."
refuseDecline a request with a reason"Cannot access that URL — outside my permissions."
failureReport that an action could not be completed"Search API returned 503 after 3 retries."
Protocol choice guide: Use direct messages for simple pipelines (<5 agents, fixed flow). Use blackboard when agents need to share state asynchronously. Use pub/sub when agents dynamically join/leave or multiple agents react to the same event. Use FIPA ACL when you need formal, auditable agent conversations.

Lab: Implement a Pub/Sub System for 3 Agents

1

Build the Pub/Sub Bus

from collections import defaultdict
import time, uuid

class PubSubBus:
    def __init__(self):
        self._subs = defaultdict(list)
        self._log = []

    def subscribe(self, topic, agent_id, callback):
        self._subs[topic].append((agent_id, callback))

    def publish(self, topic, payload, publisher):
        event = {"id": str(uuid.uuid4())[:6], "topic": topic, "payload": payload,
                 "publisher": publisher, "timestamp": time.time()}
        self._log.append(event)
        print(f"📡 [{publisher}] → {topic}: {str(payload)[:50]}")
        for agent_id, cb in self._subs[topic]:
            cb(event)

    def stats(self):
        print(f"\nBus stats: {len(self._log)} events, {len(self._subs)} topics")

bus = PubSubBus()
print("Bus ready")
2

Create 3 Specialized Agents

class ResearchAgent:
    def __init__(self, bus):
        self.bus = bus
        self.name = "ResearchAgent"
        bus.subscribe("task.research", self.name, self.on_task)

    def on_task(self, event):
        topic = event["payload"]["topic"]
        print(f"  🔍 {self.name}: researching '{topic}'")
        result = f"Found 5 key papers on {topic}: [paper1, paper2, paper3...]"
        self.bus.publish("research.complete", {"topic": topic, "result": result}, self.name)

class WriterAgent:
    def __init__(self, bus):
        self.bus = bus
        self.name = "WriterAgent"
        bus.subscribe("research.complete", self.name, self.on_research_done)

    def on_research_done(self, event):
        print(f"  ✍️  {self.name}: drafting from research on '{event['payload']['topic']}'")
        draft = f"Draft article on {event['payload']['topic']}: Introduction... Body... Conclusion..."
        self.bus.publish("draft.ready", {"draft": draft, "topic": event["payload"]["topic"]}, self.name)

class ReviewAgent:
    def __init__(self, bus):
        self.bus = bus
        self.name = "ReviewAgent"
        self.reviews = []
        bus.subscribe("draft.ready", self.name, self.on_draft)
        bus.subscribe("research.complete", self.name, self.on_research_done)

    def on_draft(self, event):
        print(f"  🔍 {self.name}: reviewing draft on '{event['payload']['topic']}'")
        self.reviews.append({"topic": event["payload"]["topic"], "verdict": "approved", "score": 8})
        self.bus.publish("review.complete", {"verdict": "approved", "score": 8}, self.name)

    def on_research_done(self, event):
        print(f"  📋 {self.name}: noting research completion for '{event['payload']['topic']}'")

researcher = ResearchAgent(bus)
writer = WriterAgent(bus)
reviewer = ReviewAgent(bus)
print("All agents registered")

class SEOAgent:
    """Dynamically added agent — subscribes to draft.ready."""
    def __init__(self, bus):
        self.bus = bus
        self.name = "SEOAgent"
        bus.subscribe("draft.ready", self.name, self.on_draft)
        print(f"  🆕 {self.name} joined the bus dynamically")

    def on_draft(self, event):
        print(f"  🔎 {self.name}: analysing SEO for '{event['payload']['topic']}'")
        keywords = ["agentic AI", "autonomous agents", "LLM agents", "multi-agent systems"]
        self.bus.publish("seo.analysis", {"keywords": keywords, "score": 72}, self.name)
